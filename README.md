1. Исправления в соответствии с Google Style
   
1.1 Пространство имен
   
Убрано using namespace std;

Добавлен префикс std:: ко всем стандартным типам и функциям

1.2 Имена функций

Enqueue → enqueue (lower_case_with_underscores)

Dequeue → dequeue (lower_case_with_underscores)

1.3 Имена переменных

Q → queue (snake_case)

N → element_count (snake_case)

newNode → new_node (snake_case)

1.4 Отступы и форматирование

Отступы изменены с 4 пробелов на 2 пробела

Открывающие скобки { перенесены на ту же строку

2. Исправления в структурах
   
2.1 PNode

Было: отсутствовал конструктор

Стало: добавлен конструктор PNode(int value) : data(value), next(nullptr) {}

Назначение: удобное создание узлов с инициализацией полей

2.2 TQueue (переименовано из PQueue)

Было: структура PQueue с полями front и last

Стало: структура TQueue с полями head и tail (по заданию Dynamic26)

Назначение: соответствие формулировке задания

3. Исправления в функциях

3.1 enqueue (бывшая Enqueue)

Было: создание узла через new PNode{value, nullptr}

Стало: создание узла через конструктор new PNode(value)

Назначение: использование добавленного конструктора

3.2 dequeue (бывшая Dequeue)

Было: отсутствовала проверка на пустую очередь

Стало: добавлена проверка if (queue.head == nullptr)

Назначение: предотвращение падения программы при извлечении из пустой очереди

4.1 Ввод данных с клавиатуры

4.2 Проверки ввода

Проверка на element_count ≥ 5 (согласно заданию)
